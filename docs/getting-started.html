Getting Started Last updated: 2023-02-11

[Project] is a project management multi-platform app allowing to create project plans, schedules, timelines, roadmaps.

What you can do with [Project]
  • plan and estimate your project by splitting the project into a number of tasks and setting dependencies between them
  • review you plan using Gantt chart
  • manage project resources (work, material or cost) and assign them to tasks
  • manage project calendars (set working times, add holidays etc)
  • manage resource calendars (working times, vacations etc)
  • plan and estimate costs
  • control your project's progress by setting % complete values
  • review critical tasks
  • be notified if a resource is overallocated
  • auto-level overallocated resources
  • mark tasks as milestones
  • set task deadlines and be notified if the task is scheduled to finish later
  • set task constraints when you need full control over tasks' start or finish dates
  • set lag time and lead time between tasks.
Compatibility

[Project] is a user-friendly alternative to Microsoft Project, if you're familiar with Microsoft Project, you can quickly get started with [Project].

[Project] can open files saved by Microsoft Project in XML format. Files saved by [Project] also use the same XML format, so they can be opened by Microsoft Project or software compatible with it.

Cloud support

[Project] can can create, open and save local files on your device. If you choose, you can sign in to your Google account to be able to open and save files on your Google Drive.

Platform support

[Project] is available on Windows, Android and the Web.

Installation

[Project] is multi-platform, so install the version that is right for you device.

  • Windows:Get it at Microsoft Store
  • Android:Get it at Google Store
  • Web:Just open this link

Note

[Project] is currently in Open Beta status. This means that you can get the app for free, yet some bugs are possible. We would like to hear your feedback!

Logging in (optional)

Once installed, you can start working with [Project] right away by creating, opening and saving local files on your device.

If you would like to be able to have your files in the cloud, you can Sign in to Google. This gives you the ability to open and save files to your Google Drive.

Note

[Project] requires permission to only be able to edit the files that it creates. No other files are read or written, and the permission for them is not requested.

Creating basic schedule

Let's say you have a small home renovation project of 2 tasks, one is done after another.

Once you click on Create New project button, you're brought to a split screen containing your future list of tasks on the left and your Gantt chart on the right.

Add your tasks one after another by clicking on the big "+" button and specifying their names and predicted durations.

Select both of your tasks by clicking on them in the list on the left and click "Link" tasks button. This makes them connected and the Gantt chart is updated.

Now that the tasks are linked if you decide to update the duration of the first task (by double clicking it in the Gantt chart and editing Duration field in Task Properties dialog), you can see how this change affects the schedule shifting the second task.

Setting project's Start Date and Name

Your project might start on a different day than the default used by [Project].

Click on the menu icon, and then on Project Properties.

Set the new project date in Project Start Date field.

Also, don't forget to change the name of the project while you're there.

Note

With all the simplicity of the basic schedule, by looking at the Gantt chart you can already tell when the project is expected to end, which is the answer to one of the most important questions in project management.

Adding and assigning resources

Let's say that 2 different people will do each of the tasks in your project.

Click on the navigation menu button and swith to Resources. Click on the big "+" button and add 2 work resources to your project: Painter and Carpenter.

Using the navigation menu switch back to Tasks, double click your first task in the Gantt chart to open Task properties.

Switch to Resources tab, check Painter and save your changes.

Repeat the same steps for your second task and the second resource: Carpenter.

Setting vacations

People can have different events during the course of your project, affecting its timeline. Let's say that one of the resources in your home renovation project, Painter, plans to have 1 day off during it.

To configure this, use navigation menu to go to Resources.

Double click on Painter so see Resource Properties. Go to Calendar tab and click "Add exception" button. Choose one of the days during which the Painter is supposed to work, leave the list of working times empty, and save all your changes.

Using the navigation menu go back to Tasks and see how your project has changed and the task following the Painter's task has also shifted due to its dependency on it.

Note

Similarly you can configure not only vacations, but also other changes to a person's schedule. For example, by adding working time to the Exception you can tell [Project] that on that particular day the resource works only during specified hours.

Setting holidays

Let's say that there's a holiday during the course of your project, a day during which all your resources are off.

While vacations are configured through each resource's calendar, days affecting everyone are configured in project's calendar. The default project's calendar is called Standard and it already contains information that no work is done on weekends.

To configure this, use navigation menu to go to Calendars.

Double click on Standard so see Calendar Properties. Click "Add exception" button. Choose one of the days during the course of your project, leave the list of working times empty, and save all your changes.

Using the navigation menu go back to Tasks and see how your project has changed and the tasks are shifted. In the Gantt chart your holiday is shown similarly to the weekends.

Note

Similarly you can configure not only holidays, but also other changes to everyone's schedule. For example, by adding working time to the Exception you can tell [Project] that on that particular day everyone works only during specified hours.

You can also take a look at tabs representing each weekday in Calendar Properties dialog. If your team, for example, leaves earlier on Fridays, you can configure this on the corresponding tab.

Saving your project

Any time you can save your project by clicking on Save button in the main toolbar.

Your project can be either saved as a local file on your device or in Google Drive.

Once your project is saved and if autosave is enabled in your options, [Project] autosaves every change you make to the file to the destination of your initial choice: local or Google Drive.

Your project is always saved in a format that is fully compatible with Microsoft Project.
