Tasks Last updated: 2023-02-11

Deadlines

Sometimes you need to keep in mind that a task should be complete by a particular day. It's important to remember about the day, so [Project] has an easy to use interface to assign deadlines to tasks.

In "Task Properties" dialog go to the first tab (General). Choose your deadline date in "Deadline".

Deadlines are for your own reference only, they do not affect scheduling.

Deadlines are shown in the Gantt chart as red circles.

Note

If your schedule shows that a task finishes later than its deadline specifies, [Project] shows an icon in the list of tasks and counts such tasks in the navigation drawer.

Note

You can set a deadline for the entire project by setting the deadline to its root summary task.

Milestones

Any task can be marked as a milestone by ticking the appropriate flag in "Task Properties" dialog. This doesn't change its duration, doesn't affect scheduling, but the task is shown in the Gantt chart as a black circle.

It is possible to create a milestone as a task of 0 duration.

Critical Tasks

Depending on your schedule, some tasks can be implemented with some delay but this will not make the entire project longer. But if some other tasks are delayed the entire project will be delayed. Such tasks which cannot be delayed without delaying the entire project are called critical tasks. If you expect your project to finish in time it is usually important to pay attention to critical tasks.

[Project] allows to detect critical tasks automatically. In the main menu choose "Highlight critical tasks", such tasks are highlighted in red color in the Gantt chart.

Task Constraints

In combination with task dependencies (tasks linked to the task as predecessors), constraint of a task defines how the task is scheduled.

Constraints are configured on "Advanced" tab of "Task Properties" dialog. The default constraint for a task is "As soon as possible" (in projects planned from Finish date the default constraint is "As late as possible"). This means that the task is set as close to the project's start date as possible with respect to dependencies with other tasks the task has.

When you need to define that a task needs to start on a particular date, it is recommended to set "Start to earlier than" (followed by the chosen date) constraint. When you need to define that a task needs to finish on a particular date, it is recommended to set "Finish no earlier than" (chosen date) constraint. These constraints are called flexible, as they respect task dependencies and if such dependencies cause the task to start/finish at a later date then the task starts/finishes at the later date. Other 2 flexible constraints are "Start no later than" and "Finish no later than" which can be configured at "Advanced" tab of "Task Properties" dialog.

But there are also 2 constraints which force the task to start or finish at the date specified regardless of dependencies. These constraints are called inflexible constraints and are "Must start on" and "Must finish on". It is recommended to set such constraints to your tasks only if you are absolutely confident that they are necessary for these tasks.

Task Constraints:
As soon as possible Task is scheduled as soon as the predecessors allow. So, if no predecessors linked, the task starts at the beginning of the project.
As late as possible Task is scheduled as late as the predecessors allow. So, if no predecessors linked, the task finishes at the end of the project.
Start no earlier than If the task starts later than the specified date due to predecessors, nothing changes. Otherwise the task is scheduled to start on the specified date.
Start no later than If the task starts earlier than the specified date due to predecessors, nothing changes. Otherwise the task is scheduled to start on the specified date.
Finish no earlier than If the task finishes later than the specified date due to predecessors, nothing changes. Otherwise the task is scheduled to finish on the specified date.
Finish no later than If the task finishes earlier than the specified date due to predecessors, nothing changes. Otherwise the task is scheduled to finish on the specified date.
Must start on Task's start date is scheduled to be exactly as specified, regardless of predecessors.
Must finish on Task's finish date is scheduled to be exactly as specified, regardless of predecessors.

Task Type

Task's Type does not affect how the task is scheduled.

Task Types:
Fixed units Task is scheduled as soon as the predecessors allow. So, if no predecessors linked, the task starts at the beginning of the project.
Fixed duration Task is scheduled as late as the predecessors allow. So, if no predecessors linked, the task finishes at the end of the project.
Fixed work If the task starts later than the specified date due to predecessors, nothing changes. Otherwise the task is scheduled to start on the specified date.
